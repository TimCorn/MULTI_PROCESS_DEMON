


                     Краткое описание
                    ------------------
  
Демон и клиент разработаны на базе Boost Asio асинхронного многопроцессного 
сервера. Требуется Boost не ниже v. 1.67. Все запускалось в Debian 10.
  
Демон AServerDemon запускается с одним параметром: номер порта.
Сервер многопроцессный - на каждого подключенного клиента свой процесс.
По номеру pid такого процесса сервер создает файл с расширением *.dat,
который он получает по сети от клиента.
Все созданные файлы располагаются в папке "SERVER_FILES" каталога проекта.

Клиенты SClient запускаются с тремя параметрами: адрес сервера, порт,
путь к файлу который передается. Все передаваемые файлы следует располагать
в папке "CLIENT_FILES".
В примере запуска скриптом RUN.sh передаваемые файлы располагаются в папке 
"CLIENT_FILES" каталога проекта.

Пример запуска сервера:
  
  ./AServerDemon  12345
  
  
Если выскакивает сообщение:

     Exception: bind: Address already in use
     
Нужно поменять порт.   


Пример запуска клиента:

  ./SClient localhost  12345 CLIENT_FILES/infile_1.txt
  

Сервер завершает свою работу по сигналам:

 pkill -15 AServerDemon    //  SIGTERM 
  
 pkill -1  AServerDemon    //  SIGHUP       
  
  
Для отладки:


RUN.sh                 - скрипт для ручного запуска демона и 5-ти клиентов.
                         Когда все клиентские окна закроются, это значит что
                         перекачка файлов закончена.
        
                         
SLAY.sh                - скрипт чтобы удалить демона запущенного скриптом RUN.sh .
                         

                         
CLIENTS_START.sh       - скрипт для 5-и клиентов SClient.

PROCESS_LIST.sh        - показывает запущенные экземпляры демона.
 
 
Запускать все  указанные утилиты можно из каталога buld. Соответственно, если 
вносились изменения в конфигурационные файлы, то после пересборки эти изменения
будут уничтожены, потому что базовые настройки конфигураций хранятся в каталоге src 
и во время сборки они автоматически копируются в каталог build.
Об этом следует помнить.


Чтобы собрать:

cd build
cmake ..
make
